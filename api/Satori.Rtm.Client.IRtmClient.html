Interface IRtmClient

The IRtmClient interface is the main entry point for accessing the RTM service, including publish and subscribe operations.

Inherited Members
IDispatchObject.Dispatcher
Namespace: Satori.Rtm.Client
Assembly: Satori.RTM.Net45.dll
Syntax
public interface IRtmClient : IDispatchObject
Remarks

Create an instance of the client with RtmClientBuilder and use the IRtmClient methods to start, stop, and restart the client WebSocket connection, and publish and subscribe to channels.

Delete(String)

Deletes a value from the specified channel.

Declaration
Task<RtmDeleteReply> Delete(string channel)
Parameters
Type Name Description
System.String channel

The channel represents a key
Returns
Type Description
System.Threading.Tasks.Task<RtmDeleteReply>

The task which contains the successful reply or exception. Task completes successfully when an acknowledgement from the server is received.

Remarks

This method is provided for key-value (dictionary storage) semantics to erase a value for a given key. Key is represented by a channel, and only the last message in the channel is relevant (represents the value). Hence, publishing a null value, serves as deletion of the the previous value (if any).

If client is not connected to the RTM service, the read request is queued. The SDK reads the value when the connection is established. The length of this queue is limited by PendingActionQueueLength. If the queue is full, the QueueFullException exception is returned in the task.

Dispose()

Disposes the client.

Declaration
Task Dispose()
Returns
Type Description
System.Threading.Tasks.Task

The task completes when this call is executed on the Dispatcher.

Remarks

Call Dispose() when you are finished using the IRtmClient. The Dispose() method leaves the IRtmClient in an unusable state. After calling Dispose(), you must release all references to the IRtmClient so the garbage collector can reclaim the memory that the IRtmClient was occupying.

Type Parameters
Name Description
T

It can be any reference or value type. The Newtonsoft.Json library is used for deserialization.

Remarks

This method is provided for key-value (dictionary storage) semantics: RtmReadRequest.Channel name represents a key and the last (and the only used) message the channel represents a value. In other words, a channel serves as a dictionary entry.

If client is not connected to the RTM service, the read request is queued. The SDK reads the value when the connection is established. The length of this queue is limited by PendingActionQueueLength. If the queue is full, the QueueFullException exception is returned in the task.

Start()

Starts the client. Client tries to establish the WebSocket connection to the RTM service asynchronously.

Declaration
void Start()
Remarks

The SDK attempts to reconnect to the RTM service if the WebSocket connection fails for any reason. If you make any publish or subscribe requests while the WebSocket connection is not active, the SDK queues the requests and completes them when the connection is established or re-established. You can use the client events to define application functionality for when the application enters or leaves the connected state.

Stop()

Stops the client. The SDK tries to close the WebSocket connection asynchronously and does not start it again unless you call Start() .

Declaration
void Stop()
Remarks

Use this method to explicitly stop all interaction with the RTM service. You can use the client events to define application functionality for when the application enters or leaves the connected state.
